## web技術 chapter-4
## HTML
DOCTYPE宣言はHTMLのバージョンを示すものだった。
## WEBページで使用される画像形式
### JPEG( Joint Photographic Experts Group )
人の目が感じにくいデータを削ることでデータサイズを小さくする形式。1677万色の色を扱える。
### GIF ( Graphics interchange Format )
データの圧縮による劣化が起こらない。複数のGIF画像をパラパラ漫画のように表示することでアニメーションにできるのが特徴。最大２５６色。
### PNG ( Portable Network Graphics )
1667万色の色が扱えて、画像の劣化が起こらない。GIFよりもデータサイズが小さい。透明度を扱える。
## XML ( Extensible Markup Language )
Webに特化したHTMLに対し、XMLは様々な用途に汎用的に利用できるマークアップ言語。目的に応じてタグを自分たちで定義することができる。
## XHTML
XMLとHTMLを合体させた記法。独自のタグが使える、定義が厳密。
## CSS
HTMLやXMLの表示方法を表現する記述。
## スクリプト言語
動的処理に必要な言語。クライアントサイド・スクリプトはJavaScript、サーバーサイド・スクリプトはPython,php,Rubyなど。
## DOM( Document Object Model )
DOMは、ブラウザでJavaScriptを使い、HTMLやXMLを操作できる様にする仕組みの事。プログラムからHTMLやXMLの各要素を参照、制御できる。文章の各要素を抽出し、階層構造として扱う。この構造をDOMツリーという。枝葉にあたる部分はノードと呼ばれ、DOMはノードを辿っていく事でデータにアクセスし参照や編集を行う。
## JSON ( Javascript Object Notation )
構造化したデータをあわらすためのデータ記述言語の一種。単なるフォーマットのこと。Javascript専用ではない。web上のデータのやり取りによく使われる。DOMを使うよりも早い。JSONでは数値、真偽値、文字列、NULL、配列、オブジェクトが表現できる。
## フィード
webサイトの更新履歴を配信するためのファイル。ハイパーリンクの集まり。RSSやAtomという形式で利用されている。
## フィードリーダー
フィードを取得し、管理するためのソフトウェア。様々なサイトの新着情報をまとめてチェックできる。RSSリーダーと呼ばれることもある。更新情報をユーザーが閲覧できるような機能が備わっている。
## ポッドキャスト
サーバー上に音楽や動画を設置しRSSを通してWEB上で公開する手法。
## マイクロフォーマット
ウェブサイトの中に意味を表現する記述を埋め込むための書式。セマンティックwebを実現できる。
## 音声・動画配信
ダウンロード配信とストリーミング配信がある。YouTubeなどはダウンロード配信方式で、ユーザー側でファイル全体をダウンロードする。ファイルがユーザー側に残るため、著作権のあるデータ配信には向いていない。ニコニコ生放送やライブ配信はストリーミング配信で、ファイルを細かく分割し細切れにしたデータをユーザーに送る。再生したデータは都度削除される。ストリーミングサーバーが必要。
## メディアタイプ
HTMLやCSSで閲覧に使うデバイスの種類を指定することができる。HTMLではヘッド内のリンクで、CSSでは@media〜で指定できる。

## RailsのMVC
ソフトウェアの構造の考え方。アプリケーション固有のデータ処理や処理の扱いの部分のモデル、UIに関わる部分のビュー、モデルとビューを統合的に制御するコントローラー。
## 処理の順番
- クライアントからのリクエストはwebサーバーを介して、railsアプリケーションへと受け渡される。
- ルーティング機能がそのリクエストの処理を行うべきコントローラーとアクションを特定し、アクションを呼び出す。
- アクションでは必要に応じてモデルを利用する。全ユーザーデータをUserクラスから取得する様に。
- モデルはデータベースの書き込みや呼び出しも担当する
- アクションごとに対応するビューテンプレートを用いてHTMLを生成する。
- コントローラーがレスポンスを作成し、webサーバーを介してクライアントに返す。

```
= form_with model: @task,lacal:true do |f|
```

## renderとredirect_to
リダイレクトはもう一度ブラウザがリクエストし直して画面を表示させる。別のurlに案内する動きをリダイレクトという。renderは対象のページを表示するのみで、render :newとした場合はnewアクションは実行されず、対象のviewを表示するのみ。redirect_to :newとすれば再度リクエストをし直すので、newアクションが実行された後、対象のviewが表示される。
## フラッシュメッセージ
redirect_toは特定のflashメッセージを渡す事ができる。セッションを利用している。
```
    redirect_to tasks_url,notice:"タスク「#{task.name}」を登録しました。"

   ↓ #同じ意味

　　flash[:notice] = "タスク「#{task.name}」を登録しました"
 redirect_to tasks_url
```
flash[:notice]にタスク完了のメッセージをセットしてからリダイレクトすると言う意味。
## フラッシュのキー設定
デフォルトで:notice、:alterが入っている。flash:の値にハッシュとして渡せばどの様なキーのデータも渡せる。
```
   redirect_to tasks_url,flash:{warning:"何かの警告メッセージ"}
```
noticeとalertはメソッドの形でもデータを渡せる
```
   flash.notice = "タスク「#{task.name}」を登録しました"
```
## フラッシュの役割
フラッシュは主に次のリクエストに対してデータを伝える仕組み。しかし同じリクエスト内で（直後にレンダーされるビュー）伝えることもできる。
```
   flash.now.alert = "登録できません"
```
2つ以上先のリクエストにデータを渡す`flash.keep`もある

## パーシャルテンプレート
異なるファイルの共通部分を1つにまとめたファイル。先頭に_をつける。呼び出す際には_を外す。
```
= render partial: 'form',locals： {task:@task}
```
`locals: {task:@task}`の意味は、パーシャルでは`＠task`などのインスタンス変数をローカル変数`task`に変更すると言う意味。変更しなくても問題ないが、使うと見やすい。
