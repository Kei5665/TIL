## クロスサイトスクリプティング(XSS)
悪意のあるリンクを表示して、そこにユーザーをアクサスさせ、悪意のあるスクリプトを実行させ、cookie情報を表示させたりやウィルスをダウンロードさせたりする。
## SQLインジェクション
フォームに悪意のあるSQLを書いて送信し、DBに意図しない操作を行わせる
## セキュリティーホール
ソフトウェアの欠陥により権限がないユーザーが権限を持つユーザーと同じ操作ができたり、見えるべきでない情報が第三者に見えてしまったりする不具合のこと
## ファイヤーウォール
インターネットと内部ネットワークに設置し、送受信されるデータを監視して通信の許可、拒否を行う仕組み。ポート番号やIPアドレスを識別して通信の許可拒否を判断する。

## 現場rails chapter-5
## ドライバ
ブラウザ相当の機能を利用するために必要なプログラム
## describe
「何についての使用を記述しようとしているのか」を記述する。ファイル全体の主題。仕様書内の見出しを展開するイメージ
## context
describeのエイリアス。状況、状態のバリエーションごとに分類する。
## before
describeやcontextを実行する前に実行される。なので前提条件のコードを記述する。DRYにするためにも重要。
## it
テストの期待する結果を書く。この部分の動作通りに動作すれば成功、動作しなければ失敗となる。
## Factory Bot
1. データを作成するためのテンプレートを用意する
2. system specのboforeなどでテスト用のデータベースにテストデータを投入する。teskデータやユーザーデータなど。
## ファクトリファイルの使い方
- ファクトリファイルのfactoryメソッドに属性の内容を示すハッシュを渡す
- 関連するファクトリファイルのファイル名も記述する
## bofore
最初の引数？には作ったファクトリファイルを指定する。
```
user_a = FactoryBot.create(:user,name: 'ユーザーA',email: 'a@example.com')
# factories/users.rbを指定している
```
ユーザーオブジェクトに紐付け
```
 FactoryBot.create(:task,name:'最初のタスク',user: user_a)
```
