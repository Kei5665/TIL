## オブジェクト
オブジェクトはクラスから作られる。
## クラス
種族みたいなもの。通行人→人間、隣の家のタマ→ねこ
## インスタンス
オブジェクトXのクラスがAであるとき、XはAのインスタンス（オブジェクト）であるという。私は人間クラスのインスタンスであり、たまはねこクラスのインスタンスである。
## ローカル変数、インスタンス変数
特定の処理が終わると捨てられるローカル変数、オブジェクトが存在する限り一緒に存在するインスタンス変数
## 定数
大文字から始まる変数。保持する値が不変
## インスタンスメソッド
ex)タマ.追いかける（ネズミ）
インスタンス（タマ）に対して呼び出せるメソッド（追いかける）をインスタンスメソッドという。
## 引数
メソッドの引数に使うカッコは省略できる。代わりに半角スペースを開ける
## インスタンス変数
先頭に@をつける。インスタンス変数に代入したデータは、インスタンス変数を抱えているオブジェクト（User.newで作るやつ）が存在する限り、保持される。つまり、他のメソッド内でも使える。対してローカル変数はそのメソッド内でしか使えない。
## ゲッター、セッター
- ゲッター
```
def name=(name)
  @name = name
end
```
- セッター
```
def name
 ( return ) @name
end
```
## まとめ
ルビーのオブジェクトのデータがインスタンス変数、振る舞いが（インスタンス）メソッド。それぞれの処理がどのようであるかをクラスに定義しておいて、その定義を使って（インスタンス）オブジェクトを好きなだけ量産していく。

## initializeメソッド
.newで新しいオブジェクトを作るときは、initializeメソッドが実行される。

## メソッドの呼び出しに制限をかける。
privateと書いた行以降はprivateメソッドになる。外部からは呼び出せなくなる。

## キーワード引数
メソッドを呼び出す際に、引数がどんなものかをわかりやすくする方法。ex) person.name(full: true,with_age: false)。メソッドを定義するときはデフォルト値を引数に定義する。

## 継承
親クラスで定義したメソッドを、子クラス（サブクラス）でも使えるようにするやり方。railsではApplicationRecordという親クラスをよく目にする。
## オーバーライド、super
子クラスにおいて親クラスと同名で定義したメソッドは親クラスが上書きされる。また、同名のメソッドで親クラスのメソッドを呼び出したいときはsuperというキーワードを使う。
## モジュール
クラスと同じくメソッドのまとまり。しかしオブジェクトは生成できない。includeメソッドでクラス内でモジュールを取り込むことができる。モデュールで定義したメソッドをいろいろなクラスで使うことができる。これをミックスインという。
## nilガード
返り値がnillの時に代わりに値をいれる。|| = 10でnillの時10を返す
## ぼっち演算子
&.オブジェクトでオブジェクトがnillであってもエラーが発生しなくなる。
## %記法
％w（）で要素を文字列の配列にできる。%i()で要素をシンボルの配列にできる。（apple banana orange）などを["apple","banana","orange"]とか[:apple,:banana,:orange]などにできる。
## mapメソッドの簡略化
```
names = users.map do |user|
   user.name
end

↓簡略

names = users.map {|user| user.name}

↓さらに簡略

names = users.map(&:name)
```
