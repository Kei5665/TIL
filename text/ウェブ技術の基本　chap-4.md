## HTML
DOCTYPE宣言はHTMLのバージョンを示すものだった。
## WEBページで使用される画像形式
### JPEG( Joint Photographic Experts Group )
人の目が感じにくいデータを削ることでデータサイズを小さくする形式。1677万色の色を扱える。
### GIF ( Graphics interchange Format )
データの圧縮による劣化が起こらない。複数のGIF画像をパラパラ漫画のように表示することでアニメーションにできるのが特徴。最大２５６色。
### PNG ( Portable Network Graphics )
1667万色の色が扱えて、画像の劣化が起こらない。GIFよりもデータサイズが小さい。透明度を扱える。
## XML ( Extensible Markup Language )
Webに特化したHTMLに対し、XMLは様々な用途に汎用的に利用できるマークアップ言語。目的に応じてタグを自分たちで定義することができる。
## XHTML
XMLとHTMLを合体させた記法。独自のタグが使える、定義が厳密。
## CSS
HTMLやXMLの表示方法を表現する記述。
## スクリプト言語
動的処理に必要な言語。クライアントサイド・スクリプトはJavaScript、サーバーサイド・スクリプトはPython,php,Rubyなど。
## DOM( Document Object Model )
HTMLやXMLを扱う手法（API）。プログラムからHTMLやXMLの各要素を参照、制御できる。文章の各要素を抽出し、階層構造として扱う。この構造をDOMツリーという。枝葉にあたる部分はノードと呼ばれ、DOMはノードを辿っていく事でデータにアクセスし参照や編集を行う。
## JSON ( Javascript Object Notation )
構造化したデータをあわらすためのデータ記述言語の一種。Javascript専用ではない。データを木構造で表現できる。web上のデータのやり取りによく使われる。DOMを使うよりも早い。
## フィード
webサイトの更新履歴を配信するためのファイル。ハイパーリンクの集まり。RSSやAtomという形式で利用されている。
## フィードリーダー
フィードを取得し、管理するためのソフトウェア。様々なサイトの新着情報をまとめてチェックできる。RSSリーダーと呼ばれることもある。更新情報をユーザーが閲覧できるような機能が備わっている。
## ポッドキャスト
サーバー上に音楽や動画を設置しRSSを通してWEB上で公開する手法。
## マイクロフォーマット
ウェブサイトの中に意味を表現する記述を埋め込むための書式。セマンティックwebを実現できる。
## 音声・動画配信
ダウンロード配信とストリーミング配信がある。YouTubeなどはダウンロード配信方式で、ユーザー側でファイル全体をダウンロードする。ファイルがユーザー側に残るため、著作権のあるデータ配信には向いていない。ニコニコ生放送やライブ配信はストリーミング配信で、ファイルを細かく分割し細切れにしたデータをユーザーに送る。再生したデータは都度削除される。ストリーミングサーバーが必要。
## メディアタイプ
HTMLやCSSで閲覧に使うデバイスの種類を指定することができる。HTMLではヘッド内のリンクで、CSSでは@media〜で指定できる。
